5. دورة جديدة أصعب.

## 4) Modes

اللعبة يجب أن تدعم أكثر من مود، لكن كل مود مستقل في الأسماء والحفظ والمنطق.

### Sea Mode — مفعل حاليًا

ماب البحر هو الأولوية الحالية.

**الفكرة:**  
سفينة استراتيجية تدافع ضد سفن عدو. اللاعب يطور السفينة من الداخل، ثم يدخل موجات قتال بحرية بخمس لاينات.

**مصطلحات البحر الرسمية:**

- السفينة
- بدن السفينة
- المدافع
- المحركات
- الترسانة
- الدرع
- المخزن
- الكنوز
- الرادار / البوصلة
- الطاقم
- الذخيرة
- الطاقة
- الدورة البحرية

**ممنوع ظهور هذه المصطلحات داخل البحر:**

- القلعة
- الأبراج
- المنجم
- المملكة
- طور القلعة

بدائلها البحرية:

| ممنوع في البحر | البديل الصحيح |
|---|---|
| طور القلعة | طور بدن السفينة |
| الأبراج | المدافع |
| المنجم | المخزن / الكنوز |
| صحة القلعة | صحة السفينة |
| موجة القلعة | الدورة البحرية |

### Castle Mode — مؤجل

القلعة محفوظة لاحقًا كمملكة تايكون مستقلة، وليست جزءًا من تعديل البحر الحالي.

يجب أن يكون لها حفظ ومصطلحات وأنظمة مستقلة عند الرجوع لها.

## 5) Save System

الحفظ يجب أن يكون منفصلًا بين المودات.

### المفاتيح الحالية في localStorage

```txt
seaProgress
castleProgress
```

### seaProgress

يحفظ:

- عملات البحر.
- أفضل دورة بحرية.
- مستويات غرف السفينة.

الغرف الحالية:

```txt
helm      قمرة القيادة / بدن السفينة
cannons   المدافع
engines   المحركات
arsenal   الترسانة
shield    غرفة الدرع
storage   المخزن
radar     الرادار / البوصلة
crew      غرفة الطاقم
```

### castleProgress

موجود كحجز مستقبلي للقلعة.

لا يتم استخدامه لتشغيل البحر.

## 6) Sea Gameplay Systems

### 6.1 الساحة الخارجية

الساحة الخارجية للقتال فقط.

تعرض:

- السفينة يسار الشاشة.
- ٥ لاينات بحرية.
- سفن العدو تأتي من اليمين.
- مؤشرات صحة السفينة والدرع.
- HUD علوي صغير.
- أزرار اللاينات والقدرات تظهر أثناء الهجوم فقط.

### 6.2 داخل السفينة

عند الضغط على السفينة تفتح لوحة `داخل السفينة`.

اللوحة عبارة عن DOM overlay فوق اللعبة، لأنها أوضح للجوال والنصوص والبطاقات.

تحتوي على خريطة سفينة داخلية قابلة للسحب على الجوال.

### 6.3 غرف السفينة الثمانية

| الغرفة | الدور |
|---|---|
| قمرة القيادة | بدن وتحكم وتجهيز |
| المدافع | زيادة ضرر الإطلاق اليدوي |
| المحركات | تقليل وقت تجهيز الإطلاق |
| الترسانة | زيادة الذخيرة وتحسين القدرات |
| غرفة الدرع | حماية طوارئ ودرع إضافي |
| المخزن | تحسين مكافآت الكنوز |
| الرادار / البوصلة | تحسين الكشف والتوجيه ووقت الهجوم |
| غرفة الطاقم | طاقة ومساندة وإصلاح |

### 6.4 القتال

- القتال يعمل بخمس لاينات.
- العدو يأتي من اليمين.
- اللاعب يطلق يدويًا من أزرار اللاينات 1 إلى 5.
- الذخيرة محدودة حسب الدورة.
- السفن التي تصل إلى اليسار تسبب ضررًا للسفينة.
- إذا انتهى الوقت واللاعب صامد، تنتهي الدورة بنجاح.
- إذا انتهت صحة السفينة، يحصل اللاعب على مكافأة جزئية وتبدأ دورة تجهيز جديدة.

### 6.5 القدرات

القدرات مؤقتة وليست أوتوماتيك دائم من البداية.

| القدرة | القيد | التأثير |
|---|---|---|
| وابل مدافع | cooldown + طاقة + عدد محدود | إطلاق على كل اللاينات |
| مساعد إطلاق مؤقت | cooldown + طاقة | إطلاق مساعد لفترة قصيرة |
| درع طوارئ | cooldown + طاقة + عدد محدود | استعادة جزء من الدرع |
| تسريع محركات | cooldown + طاقة | تسريع الإطلاق ورفع الضرر مؤقتًا |

### 6.6 الكنوز

الكنوز تعمل كميني قيمز بحرية.

الميني قيمز الحالية:

- اختيار صندوق كنز.
- توجيه بوصلة.
- ربط حبال.

تعطي مكافآت حسب الأداء والدورة ومستوى المخزن.

## 7) UI / UX Rules

### المبادئ

- الجوال العمودي أولًا.
- الساحة الخارجية للقتال فقط.
- التطوير يتم داخل السفينة وليس فوق الساحة.
- استخدام بطاقات واضحة بدل زر ترقية واحد غامض.
- HUD علوي صغير.
- أزرار الهجوم تظهر وقت الهجوم فقط.
- عدم تغطية ساحة اللعب بأزرار كثيرة.

### ستايل البحر

- داكن.
- ملحمي.
- ألوان بحرية واضحة.
- أيقونات مباشرة.
- تباين عالي للجوال.

## 8) Technical Notes

### الوضع التقني الحالي

المشروع الحالي static.

لا يوجد `package.json` حاليًا في جذر المستودع، لذلك لا يوجد أمر build مؤكد مثل:

```bash
npm run build
```

إلى أن يتم إعادة إدخال source واضح، يتم التعامل مع `index.html` كنسخة التشغيل الأساسية.

### التوجه التقني المطلوب لاحقًا

عند إعادة هيكلة المشروع، يفضل إنشاء:

```txt
src/
  config/
    modes.ts
  systems/
    save.ts
    combat.ts
    upgrades.ts
  scenes/
    seaScene.ts
    castleScene.ts
  ui/
    shipInterior.ts
    hud.ts
```

والفكرة الأساسية:

- Phaser أو Canvas مسؤول عن القتال والرسم والحركة.
- DOM مسؤول عن HUD، بطاقات التطوير، الميني قيمز، والنوافذ.
- config مركزي للمودات حتى لا تختلط المصطلحات.
